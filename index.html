SSE API Dokumentacija

Grupa: Testiranje

1. TestConnection

Endpoint: /api/test-connection

Metoda: GET

Opis: Ova funkcija testira povezivanje sa bazom podataka koristeći definisani connection string.

Odgovori:

  • 200 OK: Uspešno povezivanje na bazu podataka.
  • 500 Internal Server Error: Greška prilikom povezivanja. Vraća detaljan opis greške.

Primer korišćenja:

GET /api/test-connection HTTP/1.1
Host: [domen]

Odgovor:

{
    "message": "Uspešno povezivanje na bazu podataka."
}

Grupa: Korisnik

1. Dodavanje korisnika

Endpoint: /api/korisnik

Metoda: POST

Opis: Ova funkcija dodaje novog korisnika u bazu podataka.

Telo zahteva (Body):

  • ime (string): Ime korisnika. (Obavezno)
  • prezime (string): Prezime korisnika. (Obavezno)
  • email (string): Email adresa korisnika. (Obavezno)
  • lozinka (string): Lozinka korisnika. (Obavezno)
  • uloga (string): Uloga korisnika. (Obavezno)

Odgovori:

  • 200 OK: Korisnik uspešno dodat.
  • 400 Bad Request: Greška prilikom dodavanja korisnika.

Primer korišćenja:

POST /api/korisnik HTTP/1.1
Host: [domen]
Content-Type: application/json

{
    "ime": "Marko",
    "prezime": "Marković",
    "email": "marko@example.com",
    "lozinka": "lozinka123",
    "uloga": "admin"
}

Odgovor:

{
    "message": "Korisnik uspešno dodat"
}

2. Ažuriranje korisnika

Endpoint: /api/korisnik/{id}

Metoda: PUT

Opis: Ova funkcija ažurira podatke o korisniku na osnovu njegovog ID-a.

Parametri:

  • id (int): Jedinstveni identifikator korisnika. (Obavezno)

Telo zahteva (Body):

  • ime (string): Novo ime korisnika. (Opcionalno)
  • prezime (string): Novo prezime korisnika. (Opcionalno)
  • email (string): Novi email korisnika. (Opcionalno)
  • lozinka (string): Nova lozinka korisnika. (Opcionalno)
  • uloga (string): Nova uloga korisnika. (Opcionalno)

Odgovori:

  • 200 OK: Korisnik uspešno ažuriran.
  • 400 Bad Request: Nevalidan unos.
  • 500 Internal Server Error: Greška prilikom ažuriranja korisnika.

Primer korišćenja:

PUT /api/korisnik/1 HTTP/1.1
Host: [domen]
Content-Type: application/json

{
    "ime": "NovoIme",
    "prezime": "NovoPrezime",
    "email": "noviEmail@example.com",
    "lozinka": "novaLozinka123",
    "uloga": "korisnik"
}

Odgovor:

{
    "message": "Korisnik uspešno ažuriran."
}

4. Login korisnika

Endpoint: /api/korisnik/login

Metoda: POST

Opis: Ova funkcija proverava kredencijale korisnika i vraća podatke o korisniku ako su ispravni.

Telo zahteva (Body):

  • email (string): Email adresa korisnika. (Obavezno)
  • lozinka (string): Lozinka korisnika. (Obavezno)

Odgovori:

  • 200 OK: Uspešna prijava. Vraća podatke o korisniku.
  • 401 Unauthorized: Neispravni kredencijali.

Primer korišćenja:

POST /api/korisnik/login HTTP/1.1
Host: [domen]
Content-Type: application/json

{
    "email": "marko@example.com",
    "lozinka": "lozinka123"
}

Odgovor:

{
    "id": 1,
    "ime": "Marko",
    "prezime": "Marković",
    "email": "marko@example.com",
    "uloga": "admin"
}

Grupa: Solarne Elektrane

1. Dodavanje solarne elektrane

Endpoint: /api/solarna-elektrana

Metoda: POST

Opis: Ova funkcija dodaje novu solarnu elektranu u bazu podataka.

Telo zahteva (Body):

  • nazivElektrane (string): Naziv solarne elektrane. (Obavezno)
  • nazivKlijenta (string): Naziv klijenta. (Obavezno)
  • lokacija (string): Lokacija solarne elektrane. (Obavezno)
  • pozicijaPanela (string): Pozicija panela. (Obavezno)
  • snagaAC (decimal): Snaga AC. (Obavezno)
  • snagaDC (decimal): Snaga DC. (Obavezno)
  • brojPanela (int): Broj panela. (Obavezno)
  • datumInstalacije (DateTime): Datum instalacije. (Obavezno)
  • vrstaPanela (string): Vrsta panela. (Obavezno)
  • vrstaInvertora (string): Vrsta invertora. (Obavezno)
  • klijentID (int): ID klijenta. (Obavezno)

Odgovori:

  • 200 OK: Solarna elektrana uspešno dodata.
  • 400 Bad Request: Greška prilikom dodavanja solarne elektrane.

Primer korišćenja:

POST /api/solarna-elektrana HTTP/1.1
Host: [domen]
Content-Type: application/json

{
    "nazivElektrane": "Solarna Elektrana A",
    "nazivKlijenta": "Klijent A",
    "lokacija": "Lokacija A",
    "pozicijaPanela": "Jugoistok",
    "snagaAC": "500.5",
