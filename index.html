SSE API Dokumentacija - Entitet: Klijent

Ova dokumentacija opisuje različite API rute koje omogućavaju rad sa entitetom "Klijent". U nastavku su detalji za dodavanje, ažuriranje, brisanje i dodavanje slike klijenta.

POST: Dodavanje klijenta

POST /api/klijent

Ova ruta služi za dodavanje novog klijenta u sistem.

Obavezni parametri:

Naziv parametraTipOpis
imestringIme klijenta
prezimestringPrezime klijenta
brojTelefonastringBroj telefona klijenta
emailstringEmail adresa klijenta
adresastringAdresa klijenta

Opcionalni parametri:

Naziv parametraTipOpis
nazivstringNaziv firme
zipKodstringPoštanski broj
pibstringPIB firme
maticniBrojstringMatični broj firme
firmaUlicastringUlica firme
firmaGradstringGrad firme
firmaZemljastringZemlja firme
firmaPostanskiBrojstringPoštanski broj firme
firmaOpstinastringOpština firme
kontaktImestringIme kontakt osobe
kontaktPrezimestringPrezime kontakt osobe
kontaktTitulastringTitula kontakt osobe
brojTelefona1stringDodatni broj telefona 1
brojTelefona2stringDodatni broj telefona 2
sifraDelatnostistringŠifra delatnosti firme
opisDelatnostistringOpis delatnosti firme
napomenastringNapomena o klijentu

Primer:

POST /api/klijent { "ime": "Marko", "prezime": "Marković", "brojTelefona": "060123456", "email": "marko@mail.com", "adresa": "Ulica 123", "naziv": "Marko doo", "pib": "123456789" }

POST: Dodavanje slike klijenta

POST /api/klijent/{id}/slika

Ova ruta omogućava dodavanje slike za klijenta.

Parametri:

Naziv parametraTipOpis
idintID klijenta
slikafileFajl slike (JPG, JPEG, PNG)

Primer:

POST /api/klijent/1/slika { "slika": [binary data] }

PUT: Ažuriranje klijenta

PUT /api/klijent/{id}

Ova ruta služi za ažuriranje podataka klijenta. Ažuriraju se samo polja koja nisu prazna.

Parametri:

Naziv parametraTipOpis
idintID klijenta
imestringIme klijenta
prezimestringPrezime klijenta
brojTelefonastringBroj telefona klijenta
emailstringEmail adresa klijenta
nazivstringNaziv firme
adresastringAdresa klijenta

Primer:

PUT /api/klijent/1 { "ime": "Marko", "email": "novi_mail@mail.com", "brojTelefona": "061234567" }

DELETE: Brisanje klijenta

Ova ruta briše klijenta na osnovu njegovog ID-a.

URL:

/api/klijent/{id}

Metoda:

DELETE

Parametri:

  • id (int) - ID klijenta koji treba obrisati (obavezno)

Odgovor:

  • 200 OK: "Klijent uspešno obrisan."
  • 400 Bad Request: "Greška prilikom brisanja klijenta."
  • 404 Not Found: "Klijent sa datim ID-jem ne postoji."

GET: Prikaz svih klijenata

Ova ruta vraća listu svih klijenata.

URL:

/api/klijenti

Metoda:

GET

Odgovor:

  • 200 OK: Lista svih klijenata
  • 404 Not Found: "Nema dostupnih klijenata"
